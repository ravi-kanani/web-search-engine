  <label for="ch-1">
    <input id="ch-1" type="checkbox" class="checkbox" />
  </label>
</div> we needed to wrap the checkbox with a <div> for styling purposes and, from a quick glance, there‚Äôs nothing wrong with this markup. however, the <div> and the <input> both have a class of .checkbox and there were confusing styles in the css file that styled the <div> first and then un-did those styles to fix the <input> itself. the fix for this is a pretty simple one: all we need to do is make sure that the class names are specific so that we can safely refactor any confusing css: <div class="checkbox-wrapper">
  <label for="ch-1">
    <input id="ch-1" type="checkbox" class="checkbox" />
  </label>
</div> the thing is that this work took more than a week to ship because we had to refactor a ton of checkboxes in our app to behave in the same way and make sure that they were all using the same
